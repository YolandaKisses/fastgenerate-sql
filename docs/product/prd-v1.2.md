# 企业内部 SQL 问答 Web 应用 PRD

**版本**：PRD v1.2  
**状态**：当前工程基线版  
**更新时间**：2026-04-30

## 1. 背景

企业内部已经沉淀了大量业务数据库，但日常取数仍然依赖会写 SQL 的人员，导致查询效率低、沟通成本高、响应慢。

本项目提供一款企业内部自用的 Web SQL 问答工具。用户手动选择数据源后，用自然语言提问；系统基于已同步的数据库 Schema、原始备注、本地补充备注，以及由 Hermes 生成的 Obsidian 本地知识库卡片，辅助生成只读 SQL。系统展示 SQL 并完成安全校验，只有用户手动确认后才执行。

产品聚焦“数据源内 SQL 问答闭环”，不做通用聊天、不做 BI 平台、不做自动执行。

## 2. 目标

- 用户可在 Web 应用中手动选择一个数据源，并输入自然语言问题。
- 维护人员可配置 `MySQL / PostgreSQL / Oracle` 数据源，并同步 Schema 元数据。
- 系统支持区分“数据库原始备注”和“本地补充备注”。
- 系统可将当前数据源的 Schema 与备注同步为本地 Obsidian 知识库卡片。
- 系统通过本地 Hermes CLI 检索和理解 Obsidian 知识库，返回澄清问题或 SQL 候选。
- 系统展示生成 SQL 与校验状态，由用户手动确认后再执行。
- 系统执行 SQL 后返回表格结果。
- 系统在本地保留问答、SQL、执行状态、耗时、错误信息和知识库使用信息，便于回看和排查。
- 系统提供本地运行设置，用于配置 Hermes CLI 路径和 Obsidian 知识库根目录。

## 3. 非目标

- 不做登录。
- 不做 workspace、多租户、RBAC 权限体系。
- 不做自动推荐数据源。
- 不做跨数据源联合查询。
- 不做图表为主的分析体验。
- 不做数据库写入、DDL、DML 操作。
- 不做外部 OpenAI 兼容模型配置。
- 不做 embedding 检索。
- 不做自动执行 SQL。
- 不做集中化审计平台和日志导出。

## 4. 目标用户

- 内部分析、运营、产品、财务等有取数需求、但不熟悉 SQL 的员工。
- 少量维护人员负责配置数据源、同步表结构、维护补充备注和同步本地知识库。

## 5. 用户场景

1. 维护人员配置“销售库”，测试连接成功。
2. 系统同步表、字段、字段类型、数据库原始备注。
3. 维护人员补充业务口径备注，并同步到 Obsidian 知识库。
4. 知识库同步成功后，用户打开工作台，手动选择“销售库”。
5. 用户输入“上周华东区销售额是多少”。
6. 系统通过 Hermes 检索当前数据源的 Obsidian 知识库。
7. 若口径不清楚，系统先返回澄清问题；用户补充后继续生成 SQL。
8. 系统返回 SQL 候选并完成只读校验。
9. 用户点击“确认并执行”。
10. 系统返回表格结果，并写入本地审计日志。

## 6. 核心流程

1. 配置本地运行设置：Hermes CLI 路径、Obsidian 知识库根目录。
2. 维护人员新增或编辑数据源。
3. 系统测试数据库连接，并展示连接状态。
4. 系统同步 Schema 元数据到本地 SQLite。
5. 维护人员维护表级和字段级补充备注。
6. 系统按数据源生成 Obsidian 知识库目录和表知识卡片。
7. 用户在工作台手动选择数据源。
8. 用户输入自然语言问题。
9. 系统校验当前数据源是否有可用知识库，并提示过期风险。
10. 系统通过 Hermes CLI 检索本地知识库并生成结构化结果。
11. 系统返回 `clarification` 或 `sql_candidate`。
12. 对 SQL 候选进行只读校验，展示校验结果。
13. 用户确认后执行 SQL。
14. 系统限制返回行数和执行时间，并展示表格结果。
15. 系统用 `audit_log_id` 绑定生成与执行，更新本地审计日志。

## 7. 范围

### 7.1 MVP 包含

- Web 应用
- 数据源配置管理
- 支持 `MySQL / PostgreSQL / Oracle`
- 数据源连接测试
- Schema 同步
- 原始备注与本地补充备注分离
- Obsidian 知识库同步
- Hermes CLI 调用
- 本地运行设置页
- 单数据源 SQL 问答
- 多轮澄清上下文
- SQL 候选展示
- SQL 只读校验
- 用户确认后执行 SQL
- 表格结果展示
- 本地审计日志页面

### 7.2 MVP 不包含

- 登录和用户体系
- 组织级权限管理
- 自动数据源推荐
- 多数据源联动
- 图表分析为主的展示模式
- embedding 检索
- 外部 LLM API 配置
- 集中化审计平台
- 高级脱敏和行级权限
- 日志导出

## 8. 边界条件

- 所有配置采用单机本地保存。
- SQLite 是本地配置、Schema、备注、任务状态和审计日志的主存储。
- Obsidian 是 Hermes 问答链路的本地知识库载体，不替代 SQLite 主存储。
- 用户必须手动确认数据源，系统不自动切换数据源。
- 工作台只展示并允许选择已通过连接测试且知识库同步成功的数据源。
- 系统永不自动执行 Hermes 生成的 SQL。
- 所有 SQL 默认限制为只读查询。
- 首版结果展示以表格为主。
- 本地审计日志仅在当前设备内查看。
- Hermes CLI 和 Obsidian 路径可在本地设置页配置。

## 9. Schema 与知识库说明

Schema 同步不调用 Hermes。系统通过数据库系统表或信息视图读取：

- 表名
- 字段名
- 字段类型
- 数据库原始表备注
- 数据库原始字段备注

用户可维护本地补充备注。补充备注与数据库原始备注分离存储，不回写数据库。

知识库同步会基于 SQLite 中的 Schema 与备注，为每个表生成 Obsidian Markdown 知识卡片。Hermes 可为表用途、核心字段、关联关系和注意事项生成辅助说明；字段明细始终来自结构化 Schema，不采用模型自由生成结果。

## 10. Hermes 问答说明

系统通过 Hermes CLI 读取当前数据源对应的 Obsidian 知识库目录。Hermes 输出必须是 JSON，且只允许：

- `clarification`
- `sql_candidate`

当问题存在歧义、统计口径不明确、表或字段无法稳定判断时，系统应优先返回澄清问题。前端保存最近对话上下文，用户回答澄清后继续生成 SQL。

## 11. 异常与状态

- Hermes CLI 不存在、不可执行、超时或返回非 JSON 时，系统必须给出明确错误。
- Obsidian 目录不存在或不可写时，知识库同步必须失败并提示原因。
- 单表知识卡片生成失败时，不中断整个同步任务。
- 知识库同步任务支持 `pending / running / completed / partial_success / failed`。
- Schema 更新晚于知识库任务完成时间时，工作台提示知识库可能已过期。
- SQL 校验失败时禁止执行。
- 日志写入失败时，主流程应提示“操作成功，但日志记录失败”。

## 12. 验收标准

- 可新增、编辑、删除并测试 `MySQL / PostgreSQL / Oracle` 数据源。
- 编辑关键连接信息后，数据源可标记为 `stale`。
- 系统可同步表结构、字段类型和数据库原始备注。
- 系统支持维护本地补充备注，并与原始备注区分展示。
- 系统可配置 Hermes CLI 路径和 Obsidian 知识库根目录。
- 系统可检测 Hermes 是否可用、Obsidian 目录是否存在且可写。
- 系统可完成知识库同步，并展示成功、部分成功和失败状态。
- 用户可手动选择已通过连接测试且知识库同步成功的数据源，并完成自然语言到 SQL 的查询闭环。
- 歧义场景先澄清，澄清后可继续生成 SQL。
- 系统能展示生成 SQL 和 SQL 校验结果。
- 系统不会自动执行生成 SQL。
- 系统能拦截非只读 SQL。
- 用户确认后，系统可返回表格结果。
- 审计日志可查看历史问题、SQL、是否澄清、执行状态、耗时和错误信息。
- 生成与执行通过 `audit_log_id` 精确绑定。

## 13. 风险点

- `Oracle` 会提高数据库适配与测试成本。
- Hermes 输出质量依赖 Obsidian 知识卡片质量、字段备注质量和业务口径清晰度。
- 本地 Hermes CLI 和 Obsidian 路径属于本机环境依赖，需要清晰的设置页和错误提示。
- 知识库过期检测目前基于同步时间，无法完全识别所有语义层面的口径变化。
- 首版不做登录，因此本地审计日志更偏操作留痕和问题排查，不是严格合规审计。

## 14. 技术栈

当前首版采用：`Vue 3 + TypeScript + Naive UI + FastAPI + SQLModel + SQLite + Hermes CLI + Obsidian`

- `Vue 3 + TypeScript`：构建本地 Web 管理与问答界面。
- `Naive UI`：承载表单、数据表格、状态标签和企业工具型界面。
- `FastAPI`：承载数据源连接、Schema 同步、知识库同步、Hermes 编排、SQL 校验、执行与日志。
- `SQLite`：保存配置、Schema、备注、知识库同步任务和审计日志。
- `Hermes CLI`：本地 SQL 生成与知识库理解入口。
- `Obsidian Markdown`：按数据源组织可读、可检索的本地知识卡片。
